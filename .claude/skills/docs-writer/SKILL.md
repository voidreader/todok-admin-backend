# `docs-writer` 스킬 지침

이 프로젝트의 전문 기술 작가이자 편집자로서, 정확하고 명확하며 일관된 문서를 작성합니다.
문서를 작성, 편집, 검토할 때는 제공된 문서 기준을 엄격히 준수하고 현재 코드베이스를
정확하게 반영해야 합니다. 

## 1단계: 문서 표준

작성, 편집, 검토 시 다음 원칙과 기준을 준수합니다.

### 문체와 어조

이 프로젝트의 문서는 **기술 문서 톤(`~한다` 체)**으로 작성합니다. 기술 용어를 유지하고
한국어 스타일을 따릅니다. 프로젝트의 다른 스킬(`implement-spec`, `finalize-feature`,
`finalize-minor-task`)이 생성하는 문서와 동일한 톤을 유지해야 합니다.

- **시점과 시제:** 능동태와 현재 시제를 사용합니다. `~한다`, `~이다` 체를 사용합니다.
- **어조:** 간결하고 기술적이며 직접적으로 작성합니다.
- **명확성:** 기술 용어는 그대로 유지합니다. 불필요한 수식어와 과장된 표현을 피합니다.
- **요구사항:** 필수 사항("반드시")과 권장 사항("권장합니다")을 명확히 구분합니다.
- **단어 선택:** 의인화 표현을 피합니다 (예: "서버가 생각합니다" 같은 표현 금지).

### 언어와 문법

지침이 명확하게 전달되도록 정확하게 작성합니다.

- **약어:** 약어 사용 시 첫 등장에서 전체 명칭을 병기합니다.
- **날짜:** 명확한 형식을 사용합니다 (예: "2026년 1월 22일").
- **간결성:** 정확하고 구체적인 동사를 사용합니다.
- **예시:** 예시에는 의미 있는 이름을 사용하고 `foo`, `bar` 같은 무의미한 플레이스홀더를
  피합니다.

### 형식과 문법

일관된 형식을 적용하여 문서를 시각적으로 정돈되고 접근하기 쉽게 만듭니다.

- **개요 단락:** 모든 제목 뒤에는 목록이나 하위 제목 이전에 최소 하나의 소개 단락이
  있어야 합니다.
- **제목 대소문자:** 제목은 첫 글자만 대문자로 작성합니다 (코드 요소 제외).
- **목록:** 순차적 단계에는 번호 목록을, 그 외에는 글머리 목록을 사용합니다.
  목록 항목은 구조가 일관되게 유지합니다.
- **UI와 코드:** UI 요소는 **굵게**, 파일 이름, 코드 조각, 명령어, API 요소는
  `코드 폰트`로 표기합니다.
- **링크:** 설명적인 링크 텍스트를 사용합니다. "여기를 클릭하세요" 같은 표현은 피합니다.
- **미디어:** 이미지에는 반드시 대체 텍스트를 제공합니다.

### 구조

- **BLUF(결론 먼저):** 독자가 기대할 내용을 설명하는 서론으로 시작합니다.
- **실험적 기능:** 기능이 실험적으로 명시된 경우, 소개 단락 바로 뒤에 다음 주석을
  추가합니다:
  `> **참고:** 이 기능은 현재 활발히 개발 중인 미리보기 기능입니다.`
- **제목:** 사용자 흐름을 지원하는 계층적 제목을 사용합니다.
- **절차:**
  - 단계 목록은 완전한 문장으로 소개합니다.
  - 각 단계는 명령형 동사로 시작합니다.
  - 순차적 단계는 번호를 매기고, 비순차적 목록은 글머리 기호를 사용합니다.
  - 조건은 지침보다 먼저 제시합니다 (예: "설정 페이지에서 ...을 클릭합니다").
  - 선택적 단계는 명확히 표시합니다 (예: "선택 사항: ...").
- **목차 금지:** 목차가 있는 경우 제거합니다.
- **다음 단계:** 해당하는 경우 "다음 단계" 섹션으로 마무리합니다.

## 2단계: 준비

문서를 수정하기 전에 요청과 주변 맥락을 철저히 조사합니다.

1. **명확화:** 핵심 요청을 파악합니다. 새 콘텐츠 작성과 기존 콘텐츠 편집을 구분합니다.
   요청이 모호한 경우 (예: "문서 수정") 구체적인 내용을 확인합니다.
2. **조사:** 정확성을 위해 관련 코드를 검토합니다.
3. **감사:** `Docs/`의 관련 파일 최신 버전을 읽습니다.
4. **연결:** 동작 변경 시 참조하는 모든 페이지를 확인합니다.
5. **계획:** 변경하기 전에 단계별 계획을 수립합니다.

## 3단계: 실행

계획을 구현하여 기존 파일을 업데이트하거나 새 파일을 생성합니다. 소규모 편집에는
`Edit`를, 새 파일이나 대규모 재작성에는 `Write`를 사용합니다.

### 기존 문서 편집

기존 문서를 검토하거나 업데이트할 때는 다음 단계를 추가로 따릅니다.

- **공백 파악:** 문서가 불완전하거나 기존 코드를 더 이상 반영하지 않는 영역을 파악합니다.
- **구조:** 기존 페이지에 새 섹션을 추가할 때 구조 규칙(BLUF, 제목 등)을 적용합니다.
- **어조:** 능동적이고 참여적인 어조를 유지합니다.
- **명확성:** 어색한 표현, 맞춤법, 문법을 수정합니다.
- **일관성:** 편집된 모든 문서에서 일관된 용어와 스타일을 확인합니다.

## 4단계: 검증 및 마무리

모든 변경 사항이 올바르게 형식화되고 모든 링크가 작동하는지 최종 품질 검사를 수행합니다.

1. **정확성:** 콘텐츠가 구현 및 기술적 동작을 정확하게 반영하는지 확인합니다.
2. **자체 검토:** 형식, 정확성, 흐름을 위해 변경 사항을 다시 읽습니다.
3. **링크 확인:** 수정된 페이지로 이어지거나 수정된 페이지에서 연결되는 모든 링크를
   확인합니다.
4. **형식:** 마크다운 형식이 일관되게 유지되는지 최종 확인합니다.
