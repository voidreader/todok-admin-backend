# milestone-runner 파이프라인 설정

이 파일은 `milestone-runner` 스킬이 진행할 **단계 구성**을 정의한다.
프로젝트는 이 템플릿을 자기 파이프라인에 맞게 채운다. milestone-runner는 시작 시
이 파일을 읽어 단계·경로·호출 스킬을 결정한다.

아래 네 섹션을 모두 채운다. 각 섹션 끝에 band-of-mercenaries(게임 콘텐츠
파이프라인)를 채운 **예시**를 함께 둔다. 작성 후 예시 블록은 지워도 된다.

---

## 1. 단계 목록

파이프라인의 각 단계를 순서대로 정의한다. 단계는 선형으로 진행된다(병렬·분기 없음).

각 단계에 다음을 적는다:

- **이름**: 단계의 역할을 나타내는 짧은 이름
- **호출 스킬**: 사용자가 실행할 명령어 형식. `{인자}`/`{입력경로}` 같은 자리표시자 사용
- **산출물 디렉토리**: 이 단계 산출물이 저장되는 경로
- **선행 입력**: 이 단계가 입력으로 받는 이전 단계 산출물 (없으면 "없음")

```
### 단계 1
- 이름:
- 호출 스킬:
- 산출물 디렉토리:
- 선행 입력:

### 단계 2
- 이름:
- 호출 스킬:
- 산출물 디렉토리:
- 선행 입력:
```

## 2. 작업 단위(unit) 소스

`/milestone-runner {unit}`의 `{unit}`이 무엇이고, 그 요구사항을 어디서 읽는지 정의한다.

- **소스 문서**: 작업 단위 목록을 담은 문서 경로 (없으면 "없음 — 대화로 정의")
- **단위 식별 방법**: 소스 문서에서 `{unit}`에 해당하는 부분을 찾는 방법

```
- 소스 문서:
- 단위 식별 방법:
```

## 4. 단계 스킵 정책 (선택)

특정 작업 단위에서 특정 단계를 스킵할 조건이 있으면 적는다. 없으면 비워 둔다.
실제 스킵 여부는 2단계 계획 제안에서 사용자가 최종 결정한다.

```
- (예: "벌크 데이터가 불필요한 단위는 단계 3 스킵 권장")
```

## 5. 완료 후 권장 액션 (선택)

마지막 단계 완료 후 사용자에게 안내할 다음 행동이 있으면 적는다.

```
- (예: "생성된 명세서를 /implement-spec 또는 /implement-agent로 구현")
```
