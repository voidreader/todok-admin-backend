# 서비스 이용 통계 대시보드 기획서

이 문서는 Todok 어드민 백엔드의 첫 서비스 이용 통계 대시보드 범위를 정의한다. 첫 버전은 운영자가 매일 확인하는 현황판에 집중하며, 오늘의 핵심 지표와 최근 30일 흐름을 빠르게 파악하는 것을 목표로 한다.

## 배경과 문제

Todok 어드민은 현재 서비스 이용 현황을 한곳에서 확인하는 통계 화면을 제공하지 않는다. 운영자는 서비스가 매일 얼마나 사용되는지, 메시지와 답변이 얼마나 생성되는지, 어떤 유저가 활발히 활동하는지를 빠르게 확인해야 한다.

이 대시보드는 운영자가 서비스 성장과 이상 징후를 초기에 파악하도록 돕는다. 특히 로그인 기반 DAU, 일일 메시지 발송량, 답변 작성량, 유저별 활동량을 첫 화면에서 확인할 수 있어야 한다.

## 목표와 성공 기준

첫 버전의 목표는 "매일 보는 운영 현황판"을 제공하는 것이다. 어드민이 대시보드 첫 화면에서 오늘의 핵심 수치와 최근 30일 흐름을 확인할 수 있으면 성공으로 본다.

성공 기준은 다음과 같다.

- 어드민이 오늘의 로그인 기반 DAU, 메시지 발송 건수, 답변 작성 건수를 상단 요약 카드에서 확인한다.
- 어드민이 최근 30일의 DAU, 메시지 발송, 답변 작성 추이를 확인한다.
- 어드민이 최근 30일 기준 유저별 받은 메시지 TOP 10, 보낸 메시지 TOP 10, 답변 작성 TOP 10을 확인한다.
- 보낸 메시지 수와 로그인 기반 DAU는 새 이벤트 적재 이후부터 정확하게 집계한다.

## 대상 사용자

대상 사용자는 Todok 운영자와 관리자다. 이들은 서비스 전체 사용량을 점검하고, 활동이 급증하거나 감소한 구간을 확인하며, 활동량이 높은 유저를 파악한다.

대시보드는 일반 사용자를 위한 분석 도구가 아니다. 운영자가 빠르게 스캔하고 다음 운영 판단으로 이어갈 수 있는 백오피스 화면이다.

## 핵심 기능과 범위

첫 버전은 A안인 핵심 지표 요약형으로 구성한다. 화면은 상단 요약 카드, 최근 30일 추이, TOP 10 테이블 영역으로 나뉜다.

상단 요약 카드는 오늘 기준 지표를 보여준다.

- 오늘 로그인 기반 DAU
- 오늘 메시지 발송 건수
- 오늘 답변 작성 건수

최근 30일 추이는 일자별 흐름을 보여준다.

- 로그인 기반 DAU
- 메시지 발송 건수
- 답변 작성 건수

TOP 10 테이블은 최근 30일 기준으로 보여준다.

- 유저별 받은 메시지 TOP 10
- 유저별 보낸 메시지 TOP 10
- 유저별 답변 작성 TOP 10

유저 식별 정보는 프로필 닉네임과 슬러그를 사용한다. 운영자가 실제 서비스 맥락에서 사용자를 빠르게 알아볼 수 있게 하기 위한 결정이다.

## 지표 정의

로그인 기반 DAU는 로그인 성공 이벤트를 기준으로 하루 동안 로그인한 고유 유저 수로 정의한다. 기존 Supabase Auth의 마지막 로그인 시각만으로는 과거 일별 추이를 정확히 복원하기 어렵기 때문에, 별도 로그인 이벤트 적재를 전제로 한다.

메시지 발송 건수는 서비스에서 생성된 메시지 수로 정의한다. 받은 메시지 수는 메시지가 도착한 프로필의 유저를 기준으로 집계한다.

유저별 보낸 메시지 수는 로그인한 유저가 메시지를 보낼 때 별도 발송 이벤트를 남겨 집계한다. 기존 메시지 테이블에는 로그인 발신자 식별자가 없으므로, 첫 버전에서 정확한 집계를 위해 새 이벤트 수집이 필요하다.

답변 작성 건수는 유저가 메시지에 답변을 저장한 건수로 정의한다. 유저별 답변 작성 순위는 답변을 작성한 프로필 소유자를 기준으로 집계한다.

## 비범위

첫 버전에서는 다음 기능을 제외한다.

- 코호트 분석
- 리텐션 분석
- 방문자 기반 DAU
- 로그인하지 않은 익명 방문자 분석
- 차트 커스터마이징
- CSV 다운로드
- 실시간 갱신
- 메시지 본문 노출
- 발신자와 수신자의 개별 관계 노출

보낸 메시지 수는 집계 목적의 이벤트로만 다룬다. 대시보드는 개별 메시지 본문이나 특정 발신자와 수신자의 직접 관계를 노출하지 않는다.

## 사용자 흐름

어드민은 로그인 후 통계 대시보드에 진입한다. 먼저 상단 요약 카드에서 오늘의 DAU, 메시지 발송 건수, 답변 작성 건수를 확인한다.

그다음 최근 30일 추이 차트에서 서비스 사용량이 증가하거나 감소하는 구간을 확인한다. 필요하면 하단 TOP 10 테이블에서 최근 30일 동안 받은 메시지가 많은 유저, 보낸 메시지가 많은 유저, 답변을 많이 작성한 유저를 확인한다.

## 리스크와 열린 질문

로그인 기반 DAU와 유저별 보낸 메시지 수는 기존 데이터만으로 정확한 과거 추이를 만들 수 없다. 이벤트 적재가 시작된 시점 이후부터 정확한 집계가 가능하다.

과거 구간은 기존 데이터로 계산 가능한 받은 메시지 수와 답변 작성 수만 제공하거나, 이벤트가 없는 구간으로 표시한다. 이 정책은 개발 명세 단계에서 더 구체화한다.

유저 식별 정보로 닉네임과 슬러그를 사용하므로 운영 편의성은 높지만, 어드민 화면 접근 권한과 감사 가능성을 유지해야 한다. 이 대시보드는 반드시 어드민 인증과 인가가 적용된 화면에서만 사용한다.
